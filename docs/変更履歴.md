# 変更履歴

> 機能変更を **追記式** で記録する。古いエントリは消さない。
> 1エントリ = 1日付（または1リリース）。新しいものを上に。
>
> フォーマット：
> ```
> ## YYYY-MM-DD — 概要
> - **変更**: 何を変えたか
> - **理由**: なぜ変えたか（または 設計判断.md#xxx 参照）
> - **影響範囲**: 触ったファイル / 画面
> - **関連**: コミット / Issue / やること一覧.md の項目
> ```

---

## 2026-04-08 — ドキュメント体制の整備

- **変更**: `apps2-2/docs/` 配下に 仕様書 / 変更履歴 / 設計判断 / やること一覧 の4文書を整備
- **理由**: ブレーカー選定ツールの改善フィードバックが多数発生しており、修正履歴と現状仕様を分離して追えるようにするため
- **影響範囲**: ドキュメントのみ（コード変更なし）
- **関連**: `やること一覧.md` の全項目
