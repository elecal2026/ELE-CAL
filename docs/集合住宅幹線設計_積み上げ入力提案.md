# 集合住宅幹線設計ツール 改善提案 — 容量グループ積み上げ方式

> 作成日: 2026-06-03  
> 対象: ELE-CAL `/apartment_main`  
> 検討目的: 集合住宅幹線設計ツールを、現場利用者が満足しやすい入力・出力にするための仕様提案  
> 基本方針: 計算の参照元は内線規程 第14版 JEAC 8001-2022 の集合住宅関連資料に限定する。ただし画面上では「準拠」とは表示せず、ELE-CAL全体の方針どおり参考値として扱う。

---

## 1. 結論

集合住宅幹線設計ツールは、戸数だけを入力する「表引き専用ツール」ではなく、**容量グループを積み上げる入力方式**を主役にするのがよい。

推奨する主入力は次の形。

| 住戸容量 | 戸数 | 備考 |
|---:|---:|---|
| 30A相当 | 10戸 | 小さめ住戸 |
| 40A相当 | 5戸 | 標準住戸 |
| 60A相当 | 5戸 | 大きめ住戸 |

この入力から住戸容量を集計し、戸数に応じた集合住宅の需要率を掛けて、想定最大負荷・幹線電流・主開閉器候補を表示する。

ただし、内線規程の集合住宅表は「40Aが何戸、60Aが何戸」という表ではない。標準的な戸当たり負荷を前提に、戸数ごとの需要率、想定最大負荷、幹線電流、遮断器、CVTを示す表である。そのため、A入力による積み上げ値は「表そのもの」ではなく、**入力条件に基づく計算値**として扱う。

同じ結果画面に、内線規程表の標準表値も参考表示する。

```text
入力容量からの計算値
- 住戸合計容量: 80.0 kVA
- 適用需要率: 54%
- 需要率適用後: 43.2 kVA
- 幹線電流: 216 A

参考: 内線規程表の標準値
- 20戸: 43.2 kVA / 216 A / 225 A / CVT 60 mm2
```

この形にすると、戸数だけの物足りなさを避けつつ、内線規程表との関係も見える。

---

## 2. なぜ戸数だけの表引きでは弱いか

現行の表引き型は、内線規程の表を素早く見る用途としては便利である。戸数を入れるだけで、想定最大負荷、幹線電流、遮断器、CVTが表示される。

しかし、現場利用者から見ると次の不満が出やすい。

1. **住戸の容量差を表現できない**  
   実際の物件では、すべての住戸が同じ容量とは限らない。小さめ住戸、大きめ住戸、角部屋、オーナー住戸などで想定容量を分けたい場面がある。

2. **20戸以上では個別入力が重い**  
   20戸、30戸、40戸の物件で、101号室から1行ずつ入力するのは手間が大きい。現場では「40Aが20戸、60Aが10戸」のようにまとめて考える方が自然である。

3. **戸数だけだと、結果への納得感が弱い**  
   「20戸だからこの値」と表示されても、利用者が自分の物件条件を入れた感覚が薄い。容量グループを入力できると、自分の物件の条件を反映した結果として受け取りやすい。

4. **表引きと実物件条件の差が見えない**  
   表引きだけでは、入力条件が標準表値より軽いのか重いのかが分からない。標準表値との比較を出すことで、設計上の注意点が見えやすくなる。

## 3. 内線規程表の扱い

### 3.1 一般集合住宅の表は「A別の表」ではない

内線規程の資料3-6-1は、全電化を除く集合住宅の負荷想定例である。ELE-CAL内の根拠資料では、一般集合住宅の例は **4kVA/戸** を前提にした表として整理されている。

つまり、一般集合住宅表は次のような考え方で見る。

```text
戸数
  ↓
標準的な戸当たり負荷を前提にした合計負荷
  ↓
戸数ごとの総合需要率
  ↓
想定最大負荷
  ↓
単相3線式電流、配線用遮断器、CVTケーブル
```

この表は、たとえば「30A住戸が10戸、60A住戸が5戸」のような容量混在を直接扱う表ではない。

そのため、表引きモードにA入力を追加すると、もはや純粋な表引きではなくなる。A入力を入れるなら、それは積み上げ計算モードとして整理する方が分かりやすい。

### 3.2 表引きは廃止せず、結果欄の参考表示にする

表引き専用モードを大きく残すより、積み上げ計算の結果欄に「標準表値」として併記する方がよい。

理由は次のとおり。

- 戸数だけ入力する画面にすると、利用者から「これだけでは意味が薄い」と見られる可能性がある。
- とはいえ、内線規程表の値は重要なので、完全に消すべきではない。
- 積み上げ計算値と標準表値を比較できる方が、利用者にとって判断材料が増える。
- 標準表値より小さくなる場合、大きくなる場合を警告できる。

推奨表示は次のような構成。

```text
主結果: 入力容量からの計算値
補助結果: 内線規程表の標準値
比較: 入力計算値は標準表値より +10.6 kVA
```

## 4. A入力の扱い

### 4.1 内部データはkVAで持つ

入力画面ではAを選ばせるのが分かりやすいが、内部計算では **住戸容量kVA** として扱うのがよい。

理由は、Aの意味が利用者・契約方式・表現によってぶれやすいためである。

たとえば「40A」という表現は、次のように解釈が分かれうる。

| 表現 | 換算例 | 備考 |
|---|---:|---|
| 契約アンペア相当 | 40A = 4kVA | 100V換算の感覚に近い。一般集合住宅の4kVA/戸と相性がよい |
| 契約主開閉器定格 | 40A = 8kVA | 単相3線100/200Vで 200V 換算する考え方 |
| 住戸内の分岐ブレーカー | そのまま合算不可 | 幹線需要とは別概念。入力名に使わない方がよい |

この曖昧さを避けるため、画面上はAを便利なプリセットとして見せ、計算上はkVAに変換して扱う。

### 4.2 初期版では「A相当 = kVAプリセット」にする

初期版では、次のようなプリセットがよい。

| 表示 | 内部容量 |
|---:|---:|
| 30A相当 | 3.0 kVA |
| 40A相当 | 4.0 kVA |
| 50A相当 | 5.0 kVA |
| 60A相当 | 6.0 kVA |

この方式だと、一般集合住宅の標準表値である4kVA/戸と自然につながる。たとえば、40A相当が20戸なら、住戸合計は80kVAで、資料3-6-1の20戸標準表値と整合しやすい。

```text
40A相当 = 4.0 kVA
20戸合計 = 80.0 kVA
需要率54% = 43.2 kVA
```

一方で、単相3線の主開閉器定格として `A × 200V / 1000` で見たい利用者もいる可能性がある。その場合は、初期版では入れず、将来の詳細設定として検討する。

推奨する初期表示名は、単に「A」ではなく **「A相当」** とする。

避けたいラベル:

- 子ブレーカー
- 分岐ブレーカー
- 契約アンペアだけの表記
- 主開閉器Aだけの表記

推奨ラベル:

- 住戸容量
- A相当
- kVA直接入力
- 設計想定容量

### 4.3 kVA直接入力も用意する

A相当プリセットだけでは、現場によっては足りない。

そのため、容量グループの各行には次の2種類を用意する。

```text
[容量タイプ] [容量] [戸数]

例:
A相当     40A     12戸
kVA直接   7.5kVA   2戸
```

kVA直接入力を用意すると、A換算の解釈に引っ張られず、設計者が自分で決めた住戸容量を入れられる。

## 5. 入力項目の提案

### 5.1 初期版で必要な入力

初期版では、次の入力に絞るのがよい。

| 入力項目 | 必須 | 内容 | 理由 |
|---|---|---|---|
| 住宅タイプ | 必須 | 一般集合住宅 / 全電化 | 適用する内線規程資料が変わるため |
| 容量グループ | 必須 | 30A相当×10戸、40A相当×5戸など | 20戸以上でも入力しやすい |
| 配電方式 | 必須 | 単相3線を基本。三相3線は電流のみ | 現行ツールの考え方を踏襲 |
| 共用部・その他加算負荷 | 任意 | kVAで入力 | 共用負荷を住戸需要率に混ぜないため |

これ以上の項目は初期版では入れすぎない方がよい。

### 5.2 共用部負荷は1欄だけでよい

共用部は、共用灯、ポンプ、エレベーター、換気、管理室、消防設備などに分かれる。ただし、初期版でこれらを細かく入力させるとツールの目的がぼやける。

初期版では、次の1欄でよい。

```text
共用部・その他加算負荷 kVA（任意）
```

この値は、住戸需要率を掛けた後に加算する。

```text
住戸需要負荷 = 住戸合計容量 × 戸数需要率
想定最大負荷 = 住戸需要負荷 + 共用部・その他加算負荷
```

画面上では「別途算定済みの加算負荷」として扱う。共用部の詳細計算は、ブレーカー選定や電圧降下計算など既存ツールに任せる。

### 5.3 EV充電は初期版では専用入力にしない

EV充電は重要だが、初期版で専用入力にすると、同時利用率、負荷制御、台数、出力、将来増設、別契約などの論点が一気に増える。

内線規程のみを参照元にする方針であれば、EV専用の独自計算を入れるより、初期版では「共用部・その他加算負荷」に含める方が安全である。

将来版でEVを扱うなら、次のように分ける。

- EV充電器台数
- 1台あたり出力kW
- 同時利用台数または制御後上限kW
- 加算先を住戸側にするか共用側にするか

ただし、これは初期版の範囲外とする。

### 5.4 予備率は初期版では不要

予備率を入れると便利に見えるが、内線規程表値との関係が分かりにくくなる。

初期版では、予備率を自動で掛けない方がよい。必要な場合は、利用者が共用部・その他加算負荷に予備分を含めるか、kVA直接入力で調整すればよい。

将来版で入れる場合は、次のように明確に分ける。

```text
入力容量
  ↓
需要率適用
  ↓
共用部加算
  ↓
任意予備率
```

ただし、予備率を入れるなら「内線規程表値そのものではない」ことがより強くなるため、表示上の注意が必要である。

## 6. 計算ロジック案

### 6.1 一般集合住宅

容量グループ入力を次のように定義する。

```ts
type CapacityGroup = {
  label: string
  capacityKvaPerUnit: number
  units: number
}
```

計算手順:

```text
1. 戸数 N = Σ グループ戸数
2. 住戸合計容量 P_total = Σ (各グループの容量kVA × 戸数)
3. N に対応する需要率 d を内線規程資料3-6-1の表から取得
4. 住戸需要負荷 P_dwelling = P_total × d
5. 加算負荷 P_extra を加える
6. 想定最大負荷 P_max = P_dwelling + P_extra
7. 単相3線式電流 I = P_max × 1000 / 200
8. 主開閉器候補 = I以上の標準定格候補
9. 内線規程表の標準値と比較する
```

ここで重要なのは、`d` は内線規程表から取得するが、`P_total` は利用者入力値である点である。画面上では「入力容量からの計算値」と表示する。

### 6.2 標準表値との比較

同じ戸数Nについて、内線規程表の標準値も取得する。

```text
標準表値 P_standard
標準表値 I_standard
標準表値 breaker_standard
標準表値 cable_standard
```

比較表示:

```text
差分 = 入力容量からのP_max - 標準表値P_standard
```

表示パターン:

| 状態 | 表示 |
|---|---|
| 入力計算値 < 標準表値 | 入力容量からの計算値は標準表値を下回ります |
| 入力計算値 ≒ 標準表値 | 入力容量からの計算値は標準表値と近いです |
| 入力計算値 > 標準表値 | 入力容量からの計算値は標準表値を上回ります |

ここで「どちらを採用すべき」と断定しない。あくまで比較材料として表示する。

ただし、利用者に見やすくするため、結果カードでは大きい方を強調してもよい。

```text
比較上の大きい値: 54.8 kVA
```

この表現なら、ツールが最終設計を断定せず、注意喚起として機能する。

### 6.3 主開閉器候補の扱い

積み上げ計算値に対して、幹線電流以上の標準定格を候補として表示する。

ただし、内線規程の集合住宅表から直接引いた値ではない場合は、表示名を「候補」にする。

推奨表示:

```text
主開閉器候補: 225A
```

避けたい表示:

```text
推奨ブレーカー: 225A
内線規程準拠: 225A
確定値: 225A
```

### 6.4 CVT候補の扱い

CVTケーブルについては、積み上げ計算で自動確定しすぎない方がよい。

理由は、CVT選定には次の条件が関係するためである。

- 幹線電流
- 布設方式
- 周囲温度
- 条数
- 電圧降下
- こう長
- 許容電流

ELE-CALには許容電流表、電圧降下計算、配管サイズ計算、ラック選定が既にある。したがって集合住宅幹線ツールでは、CVTを最終決定するより、**標準表値の参考表示**と**他ツールへの受け渡し値**を出す方がよい。

推奨表示:

```text
CVT参考: 60 mm2（同戸数の標準表値）
```

または、入力計算値が標準表値を上回る場合:

```text
CVT参考: 標準表値を上回るため、許容電流表・電圧降下計算で確認
```

この方が、数値の過信を避けられる。

### 6.5 全電化集合住宅

全電化は一般集合住宅より慎重に扱う。

内線規程資料3-6-2では、23時一斉始動型、マイコン制御型などの条件があり、電気温水器の想定負荷や重畳率が絡む。

初期版では、全電化については次の扱いがよい。

- 表引き値を強く残す
- 容量グループ積み上げは一般集合住宅より控えめに扱う
- 全電化の容量グループ入力を許可する場合でも、結果表示で「表値との比較」を必ず出す
- 17戸以上など表で遮断器・CVT値がない範囲は、現行方針どおり自動選定しない

全電化では、A相当入力だけで実態を表すのが難しい。電気温水器、エコキュート、IHなどの扱いがあるためである。

そのため、全電化の初期版は次のどちらかがよい。

| 案 | 内容 | おすすめ度 |
|---|---|---|
| A案 | 全電化は現行どおり表値中心。積み上げは一般のみ | 高い |
| B案 | 全電化も積み上げ入力可。ただし表値比較と注意を強める | 中 |
| C案 | 全電化詳細入力を作る | 初期版では低い |

初期版ではA案が安全である。

## 7. 画面構成案

### 7.1 入力欄

左カラムは次の構成にする。

```text
基本条件
- 住宅タイプ: 一般集合住宅 / 全電化23時 / 全電化マイコン
- 配電方式: 単相3線 / 三相3線

住戸容量グループ
- 30A相当 × [  ] 戸
- 40A相当 × [  ] 戸
- 50A相当 × [  ] 戸
- 60A相当 × [  ] 戸
- + kVA直接入力を追加

加算負荷
- 共用部・その他加算負荷 [  ] kVA
```

ポイントは、最初から行追加式にしすぎないこと。

30A、40A、50A、60Aの代表行を最初から並べると、20戸以上でも入力が速い。

```text
30A相当 [ 0 ] 戸
40A相当 [ 20 ] 戸
50A相当 [ 0 ] 戸
60A相当 [ 10 ] 戸
```

この方が、「行を追加してから容量を選ぶ」より速い。

### 7.2 個別住戸入力は詳細機能にする

個別住戸入力は不要ではないが、主役にしない。

必要になる場面:

- 部屋番号別の一覧を残したい
- 住戸ごとの容量が細かく違う
- 後から帳票化したい

ただし初期画面で個別住戸入力を主役にすると、20戸以上で重い。

そのため、次のようにする。

```text
通常: 容量グループ入力
詳細: 個別住戸入力に展開
```

たとえば `40A相当 × 10戸` を押すと、詳細表示では101〜110号室のような行に展開できる。これは将来機能でよい。

### 7.3 結果欄

右カラムは、最上部に計算結果を出す。

```text
入力容量からの計算値
想定最大負荷: 54.8 kVA
幹線電流: 274 A
主開閉器候補: 300 A

参考: 同戸数の標準表値
想定最大負荷: 43.2 kVA
幹線電流: 216 A
配線用遮断器: 225 A
CVT: 60 mm2

比較
入力容量からの計算値は標準表値より +11.6 kVA 大きいです。
```

表現は「推奨」より「候補」「参考」「比較」がよい。

## 8. 他ツールとの役割分担

ELE-CALには、集合住宅幹線設計以外に次のツールがある。

| ツール | 役割 | 集合住宅幹線ツールとの関係 |
|---|---|---|
| ブレーカー選定 | 負荷電流からブレーカー定格を検討 | 集合住宅側では主開閉器候補まで。詳細な負荷条件はブレーカー選定へ渡す |
| 電圧降下計算 | 幹線・分岐の電圧降下を計算 | 集合住宅側では幹線電流を出し、こう長・電線サイズは電圧降下計算で確認 |
| 許容電流表 | 電線種別・布設条件別の許容電流参照 | CVTを確定しすぎず、許容電流表で確認できる値を渡す |
| 配管サイズ計算 | 電線管の占有率から配管サイズを選定 | 集合住宅側では配管条件を持たない。選定後のケーブルで配管サイズ計算へ |
| ラック選定 | ケーブルラック幅・荷重を簡易選定 | 集合住宅側ではラック幅や荷重を扱わない |

この分担を踏まえると、集合住宅幹線ツールが持つべき範囲は次で十分である。

```text
集合住宅幹線ツールの責任範囲
- 住戸容量の集計
- 戸数需要率の適用
- 加算負荷の足し込み
- 想定最大負荷の表示
- 幹線電流の表示
- 主開閉器候補の表示
- 標準表値との比較
- 他ツールで確認すべき項目の案内
```

逆に、次は抱え込まない方がよい。

```text
集合住宅幹線ツールで抱え込まない範囲
- 電圧降下の最終判定
- 布設条件込みの許容電流判定
- 配管サイズ選定
- ラック選定
- EV充電の詳細な同時利用率計算
- 相バランスの詳細判定
- 電力会社ごとの契約ルール
```

### Q1への返答：kVA換算方式と入力単位について

kVA換算方式についての指摘は妥当である。特に、`40A = 4.0kVA` とだけ書くと、単相3線式の主開閉器定格を `A × 200V / 1000` で見る設計者には違和感が出る可能性がある。

ただし、初期版の入力UIとしては、kVAを主表示にするより、**Aを主表示にする方が現場利用者には分かりやすい** と考える。

理由は次のとおり。

1. ユーザーが最初に想定していた入力イメージが「40Aが10戸、50Aが5戸」のような形式である。
2. 集合住宅の住戸容量は、現場会話では「30A」「40A」「60A」のようにAで表現されることが多いと考えられる。
3. 20戸以上の物件で素早く入力するには、kVA直接入力よりA選択の方が迷いが少ない。
4. 一般集合住宅の標準表値が4kVA/戸で整理されているため、40A契約相当を4kVAとして扱うと、標準表値との比較が分かりやすい。

したがって、通常画面では `3.0kVA（30A目安）` のようにAとkVAを併記しない。表示を増やしすぎると、かえって単位の意味を考えさせてしまうためである。

推奨する通常入力ラベルは次のとおり。

```text

30A  [  ] 戸
40A  [  ] 戸
50A  [  ] 戸
60A  [  ] 戸
```

または、より説明的にするなら次の表記も候補になる。

```text
住戸容量（契約A）
30A  [  ] 戸
40A  [  ] 戸
50A  [  ] 戸
60A  [  ] 戸
```

避けたい表記は次である。

- `A相当`
- `主開閉器定格`
- `子ブレーカー`
- `分岐ブレーカー`

`A相当` は曖昧で、`主開閉器定格` は `A × 200V / 1000` の解釈を誘発しやすい。`子ブレーカー` や `分岐ブレーカー` は幹線需要と別概念なので避ける。

内部計算では、初期版は次の換算を採用する。

```text
住戸容量[kVA] = 住戸契約容量[A] / 10
```

例:

```text
40A × 20戸 = 4.0kVA × 20戸 = 80.0kVA
```

この換算式は通常画面には出さず、計算根拠の詳細欄にだけ表示する。

一方、`A × 200V / 1000` で考える入力は、初期版では入れない方がよい。もし必要になった場合は、同じA入力に混ぜるのではなく、将来の詳細モードとして分ける。

```text
詳細モード案:
- 住戸契約容量A（A / 10 = kVA）
- 契約主開閉器A（A × 200V / 1000 = kVA）
- kVA直接入力
```

ただし、初期版でここまで分けると入力が重くなるため、採用しない。

結論として、Q1に対する返答は次のとおり。

```text

画面上ではAだけを表示する。

kVAや換算式は、通常画面ではなく計算根拠欄に表示する。
```

つまり、内線規程表の範囲を超える場合は、需要率以降の計算を行わない。

### 補足Q1. 契約Aの選択肢の範囲

- **4択案**：30/40/50/60A。入力が速い、迷わない。小容量住戸は対応外。
- **7択案**：10/15/20/30/40/50/60A。実際の契約と一対一対応できる。

初期版の想定利用者（設計者）に合わせてどちらにするか確認したい。かかずさんの現場感覚で決めるのが最速。

### 補足Q2. 「住戸契約容量」という用語の現場確認

「住戸契約容量」という表現が、かかずさんら電気設計の現場で自然に通じるか。別の言い方の方が馴染みがあるなら、ラベルを変えるだけなので実装後でも直せる。優先度低だが次回打合せで一言確認しておくと安心。

## 追記3：実装方針の確定メモ（2026-06-03）

### 1. 別ページではなく `/apartment_main` を上書きする

今回の改善は、別ページや `/apartment_main_v2` として作るのではなく、既存の `/apartment_main` を上書きする方針でよい。

従来の「戸数だけを入力して内線規程表を引く」画面は一掃し、新しい容量グループ積み上げ方式に置き換える。

```text
採用:
/apartment_main を新仕様で上書きする

不採用:
/apartment_main_v2 などの別ページを作る
従来の表引きUIを並列で残す
```

ただし、内線規程表の値そのものを完全に捨てるわけではない。従来UIとして残すのではなく、**入力容量からの計算値と比較するための標準表値データ**として結果欄に表示する。

```text
残すもの:
- 同戸数の標準表値
- 標準表値との差分
- CVTや遮断器の表値参考

残さないもの:
- 戸数だけで完結する旧入力UI
- 表引き専用モード
- 旧仕様と新仕様の並列運用
```

この方針により、利用者は「どちらの集合住宅幹線ツールを使えばよいか」で迷わない。保守上も、同じ目的のページを複数持たずに済む。

### 2. 補足Q1への返答：契約Aの選択肢は初期版では4択を主表示にする

住戸契約容量Aの選択肢は、初期版では **30A / 40A / 50A / 60A** を主表示にする。

理由は次のとおり。

1. 集合住宅幹線の設計支援としては、10A / 15A / 20Aの小容量住戸を主役にする可能性は低い。
2. 20戸以上の物件で素早く入力するには、選択肢を増やしすぎない方がよい。
3. 40Aが標準的な4kVA/戸と対応し、標準表値との比較が分かりやすい。
4. 必要になった場合に、後から10A / 15A / 20Aを追加するのは容易である。

初期UI案:

```text
住戸契約容量
30A  [  ] 戸
40A  [  ] 戸
50A  [  ] 戸
60A  [  ] 戸
```

10A / 15A / 20Aは初期表示には入れない。もし実装時に拡張余地を持たせるなら、固定4行の下に「その他容量を追加」を置く。

```text
+ その他容量を追加
```

ただし、Phase 1では「その他容量を追加」は必須ではない。まずは4択で入力体験を軽くすることを優先する。

### 3. 補足Q2への返答：用語は `住戸契約容量` を暫定採用する

画面上のラベルは、初期版では **住戸契約容量** を採用する。

候補の比較は次のとおり。

| 候補 | 評価 |
|---|---|
| 住戸契約容量 | 入力意図が比較的伝わりやすい。初期採用 |
| 住戸容量（契約A） | 説明的だが、少し表示が長い |
| A相当 | 曖昧なので不採用 |
| 主開閉器定格 | `A × 200V / 1000` の解釈を誘発しやすいので不採用 |
| 子ブレーカー | 幹線需要とは別概念なので不採用 |

表示スペースに余裕がある場所では、補助テキストとして次のように書く。

```text
各住戸の契約Aを戸数ごとに入力します。
```

この補助テキストにより、`住戸契約容量` という語に不慣れな利用者でも、何を入力すればよいか分かりやすくなる。

### 4. 実装着手時の確定仕様

実装に進む場合、Phase 1は次の仕様でよい。
