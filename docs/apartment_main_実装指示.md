## 0. 大原則 — 内線規程準拠

本ツールの **唯一の根拠は内線規程 第14版 JEAC8001-2022** である。`docs/根拠/apartment_main.md` はその抜粋。

### 0.1 本ツールの設計方針

1. **内線規程の表（資料 3-6-1 / 3-6-2）に明示されている値の範囲内** のみを扱う
2. 内線規程に書かれていない範囲は **「内線規程に該当データなし」と表示してエラーとし、設計者の個別判断に委ねる**
3. 数値の独自補正・独自外挿は**行わない**

### 0.2 「内線規程に書かれていない範囲」とは

- 住戸面積が 100m² 以外（表は100m²基準のみを示す）
- 一般集合住宅で戸数 41 戸以上（表は40戸まで）
- 全電化集合住宅で戸数 17 戸以上（表に行はあるが配線用遮断器・CVTケーブルの列が「—」、つまり値が示されていない）

これらは設計者が内線規程の他章 / 個別計算 / 経験で補う領域。本ツールは踏み込まない。

### 0.3 将来の拡張

「住戸面積補正」「戸数の追加」「全電化17戸以上の対応」などは、ユーザーからの要望と内線規程の追加根拠が揃ったタイミングで別途検討する。**初期リリースには含めない**。

## 1. ツールの目的・想定ユーザー

### 1.1 目的

集合住宅（マンション・アパート）を新築/改修する際の **幹線（建物全体に電気を引き込む主配電線）の設計値** を、戸数と住宅タイプから内線規程の表に基づいて一発で出す。

設計者が内線規程の表（資料 3-6-1 / 3-6-2）を毎回めくる手間を省く。

### 1.2 想定ユーザー

- 集合住宅の電気設備を設計する電気工事士・設計者
- 既に ELE-CAL の `/breaker`・`/voltage_drop_v2`・`/pipe_size` を使っている層

### 1.3 想定アウトプットの使われ方

- 幹線CVTケーブルの発注サイズ決定
- 引込開閉器（メインブレーカー）の定格選定
- 電力会社への受電容量申請の根拠資料

## 2. 入力仕様

### 2.1 必須入力

| フィールド | 型 | 選択肢 / 範囲 | デフォルト | バリデーション |
|---|---|---|---|---|
| 戸数 | number | 1〜40（一般） / 1〜16（全電化） | 10 | 整数。タイプによって上限が変わる |
| 住宅タイプ | select | 一般 / 全電化(23時一斉始動型) / 全電化(マイコン制御型) | 一般 | — |
| 配電方式 | select | 単相3線 / 三相3線 | 単相3線 | 三相は別計算（§4.3） |

### 2.2 住戸面積の扱い

**住戸面積の入力欄は設けない**。

- 内線規程の資料 3-6-1 / 3-6-2 はいずれも **住戸面積 100m² 基準** の表のみを示す
- 100m² 以外の補正方法を内線規程は一般集合住宅について明示していない（全電化には §3.2 の式があるが、表との整合が取り切れず本ツールでの採用は見送り）
- 設計者が混乱しないよう、画面に **「住戸面積 100m² 基準（内線規程 資料 3-6-1 / 3-6-2 に準拠）」** と明示する
- 100m²以外の物件に使いたい設計者は、本ツールの結果を参考値として扱い、別途内線規程他章で補正する

### 2.3 戸数の扱い

| 住宅タイプ | 対応範囲 | 範囲外の挙動 |
|---|---|---|
| 一般集合住宅 | 1〜40戸 | 0以下・41以上はエラー（「内線規程の対応範囲外」と表示） |
| 全電化（23時一斉） | 1〜16戸 | 17戸以上はエラー（同上） |
| 全電化（マイコン制御） | 1〜16戸 | 17戸以上はエラー（同上） |

**全電化16戸の上限根拠**: 内線規程 §3.3 / §3.4 の表で配線用遮断器・CVTケーブルの値が「—」（記載なし）になるのが17戸以降のため。17戸以上は内線規程として完結したデータが提示されておらず、本ツールの対応範囲外。

## 3. 出力仕様

### 3.1 メイン出力（結果カード）

すべて内線規程の表（資料 3-6-1 / 3-6-2）から直接引いた値を表示する。再計算・補正は行わない。

| 項目 | 単位 | 出典 |
|---|---|---|
| 想定最大負荷 P_max | kVA | 表の「想定最大負荷」列 |
| 適用された総合需要率 | % | 表の「総合需要率」列（一般）/「一般電力需要率」列（全電化） |
| 適用された重畳率（全電化のみ） | 倍 | 表の「重畳率 k」列 |
| 電気温水器の想定負荷（全電化のみ） | kVA | 表の「電気温水器の想定負荷」列 |
| 単相3線式 電流 | A | 表の「単相3線式電流」列 |
| 三相3線式 電流（D案） | A | `P_max × 1000 / (√3 × 200)` で算出（§4.3） |
| 配線用遮断器の定格電流 | A | 表の「配線用遮断器の定格電流」列 |
| CVTケーブル最小太さ | mm² | 表の「CVTケーブル最小太さ」列 |

### 3.2 計算根拠の表示（E案）

`<details>` で折りたたみ、開くと以下を表示:

- 入力値の確認（戸数・タイプ・配電方式）
- 適用した表の出典（例: 「内線規程 資料 3-6-1（一般集合住宅）戸数10」）
- 表から引いた各値の一覧
- 三相3線式の場合は電流算出式（数値を代入した形）と「力率1.0で算出。実機ではcosθを考慮」の注記
- 内線規程の前提（住戸面積100m²基準、基底温度40℃のCVT許容電流に基づく、など）

### 3.3 警告・エラー表示

ResultPanel 上部に出す:

- **戸数 0 以下**: エラー（「戸数は1以上で入力してください」）
- **一般 41戸以上**: エラー（「内線規程の対応範囲外（一般集合住宅は40戸まで）。設計者の個別判断が必要です」）
- **全電化 17戸以上**: エラー（「内線規程の対応範囲外（全電化集合住宅は16戸まで）。設計者の個別判断が必要です」）
- **三相3線式 選択時**: 注意表示（「三相3線式は電流のみ算出。配線用遮断器・CVTケーブルは集合住宅向け資料に値なし、別途確認が必要です。力率1.0で算出しています」）

## 4. 計算ロジック

### 4.1 単相3線式（メイン）

戸数 × 住宅タイプから **テーブル引き** で全結果を取得。

- 一般集合住宅: 根拠資料 §2.2 の表（戸数1〜40）
- 全電化（23時一斉）: 根拠資料 §3.3 の表（戸数1〜16）
- 全電化（マイコン制御）: 根拠資料 §3.4 の表（戸数1〜16）

**再計算は一切しない**。表の値をそのまま表示する。

### 4.2 住戸面積補正

**実装しない**（§0, §2.2 のとおり）。

### 4.3 三相3線式の電流計算（D案）

単相3線の `想定最大負荷 P_max [kVA]`（=表値）から:

```
I_3相 = P_max × 1000 / (√3 × 200) [A]
```

- 力率は1.0で計算（幹線電流の概算用途のため）。注記で「力率は1.0で算出。実機ではcosθを考慮」を併記
- 三相選択時は **単相3線式の電流・遮断器・CVT値は非表示にする**（混乱を避けるため）
- 三相用のケーブル太さ・遮断器の表は内線規程の元資料に集合住宅向けの記載が無いため、**三相選択時はケーブル太さ・遮断器定格は「要別途確認」と表示し、自動算出しない**。電流値のみ出す

### 4.4 数値の丸め

- 想定最大負荷: 表の値をそのまま（小数1桁が多い）
- 単相3線式電流: 表の値をそのまま（整数）
- 三相3線式電流: 計算後に四捨五入して整数
- ケーブル太さ・遮断器定格: 表の値（整数）

## 5. データ定義（コード側）

### 5.1 配置

`src/app/apartment_main/` 配下に以下を作る:

```
apartment_main/
├── page.tsx              ... 画面本体（既存のプレースホルダを置き換え）
├── types.ts              ... ApartmentInput, ApartmentResult, HousingType など
├── data.ts               ... §2.2/§3.3/§3.4 の表をTypeScriptオブジェクトで保持
├── calculations.ts       ... テーブル引き + 三相電流換算
├── validation.ts         ... 入力バリデーション（戸数範囲・タイプ整合性）
├── ResultPanel.tsx       ... 結果表示パネル
└── InputPanel.tsx        ... 入力パネル（必要なら分割。breaker の LoadEntryRow に倣う）
```

### 5.2 表データの持ち方

`data.ts` に3種類の配列として保持。例:

```ts
export type GeneralRow = {
  units: number               // 戸数
  demandRate: number          // 総合需要率(%)
  maxLoadKva: number          // 想定最大負荷(kVA)
  currentA: number            // 単相3線式電流(A)
  breakerA: number            // 配線用遮断器の定格電流(A)
  cableMm2: number            // CVTケーブル最小太さ(mm²)
}

export const GENERAL_TABLE: GeneralRow[] = [
  { units: 1, demandRate: 100, maxLoadKva: 8.0, currentA: 40, breakerA: 40, cableMm2: 14 },
  // ... 根拠資料 §2.2 の戸数1〜40を全行転記
]

export type ElectricRow = {
  units: number
  demandRate: number          // 一般電力需要率(%)
  overlapRate: number         // 重畳率 k
  heaterKva: number           // 電気温水器の想定負荷(kVA)（表は累計値）
  maxLoadKva: number          // 想定最大負荷(kVA)
  currentA: number            // 単相3線式電流(A)
  breakerA: number            // 配線用遮断器の定格電流(A)
  cableMm2: number            // CVTケーブル最小太さ(mm²)
}

export const ELECTRIC_23H_TABLE: ElectricRow[] = [
  // 根拠資料 §3.3 の戸数1〜16を全行転記（17戸以降は本ツール対象外なので含めない）
]

export const ELECTRIC_MICOM_TABLE: ElectricRow[] = [
  // 根拠資料 §3.4 の戸数1〜16を全行転記（同上）
]
```

**転記の正解は `docs/根拠/apartment_main.md` の表を唯一の真実とする**。実装時は数値を1行1行突き合わせること。

#### 転記時の注意

- 表の `breakerA` / `cableMm2` は本ツールの対応範囲内（一般1〜40、全電化1〜16）では「—」になる行は無いはず。もし「—」を見つけたら本書の対応範囲を再確認すること
- 全電化の `heaterKva` は表記が「累計値（4.4 × 戸数）」になっている。そのまま転記すればよい

### 5.3 import 経路

既存ツールに倣って `@/components/SiteHeader` などのエイリアスを使う。

## 6. UI 設計

### 6.1 全体レイアウト

`/breaker` を参考に左右2カラム（モバイルは縦積み）:

- 左: 入力パネル（戸数・タイプ・配電方式）
- 右: 結果パネル（メイン出力 + 折りたたみで根拠表示）

### 6.2 ヘッダー・カード

`SiteHeader mode="sub" title="集合住宅幹線設計"` は現状のまま使う。
`.card`, `.card-title`, `.disclaimer` の既存 globals.css クラスに揃える。

### 6.3 配色

ホーム画面の `/apartment_main` カードは赤系（`styles.red`）になっているので、結果パネルのアクセントカラーもそれに寄せる（ブレーカー選定と同系で違和感ないはず）。

### 6.4 入力UI 詳細

- 戸数: `<input type="number">` + 増減ボタン。住宅タイプによって上限が動的に変わる（一般→40、全電化→16）
- 住宅タイプ: ラジオボタン3択
- 配電方式: ラジオボタン2択（単相3線がデフォ）
- **住戸面積 100m² 基準である旨の固定表示**（入力欄ではなくバッジや小見出しとして）

### 6.5 結果UIの並び

```
┌─ メイン結果 ──────────────┐
│  想定最大負荷: ◯◯ kVA        │
│  ───────────────────────  │
│  単相3線式 電流: ◯◯ A      │
│  推奨ブレーカー: ◯◯ A       │
│  CVTケーブル最小: ◯◯ mm²    │
│  （三相選択時は単相結果を     │
│   隠し、三相電流のみ表示）    │
└──────────────────────────┘

▼ 計算根拠を表示（折りたたみ） E案
```

### 6.6 免責表示

既存の `.disclaimer` を結果パネル下に表示。文言は既存ツールと統一:

> 本ツールの計算結果は参考値です。実務では最新の規格・基準を必ずご確認ください。

## 9. テスト観点

実装後に最低限確認すること:

### 9.1 単体テスト的な数値検証

- 戸数 1, 5, 10, 25, 40 戸 × 一般集合住宅で、根拠資料 §2.2 の表の値と完全一致するか
- 戸数 1, 5, 10, 16 戸 × 全電化23時で、根拠資料 §3.3 の表の値と完全一致するか
- 戸数 1, 5, 10, 16 戸 × 全電化マイコンで、根拠資料 §3.4 の表の値と完全一致するか
- 三相3線式選択時、電流が `P_max × 1000 / (√3 × 200)` で算出され、ブレーカー/CVT が非表示になるか

### 9.2 境界テスト

- 戸数 0 以下 → エラー
- 一般 41戸 → エラー（対応範囲外）
- 全電化 17戸 → エラー（対応範囲外）
- タイプを「一般」から「全電化」に切り替えると、戸数の上限が40→16に変わり、現在の戸数がオーバーフローしていたら自動補正 or 警告が出るか

### 9.3 UI

- スマホサイズ（〜480px）で縦積みになるか
- 結果の折りたたみ（`<details>`）が開閉するか
- 課金ガードが未契約ユーザーで発動するか
- 「住戸面積100m²基準」の表示が画面上に常時出ているか

### 9.4 既存ツールとの一貫性

- ヘッダー・カード・免責の見た目が `/breaker` 等と同じ流儀か
- 警告/エラー表示の出方が `/breaker` の `ResultPanel` と揃っているか

## 10. 採用しない機能（明示）

ユーザーから「不要」と確認済み:

- **既存ツール連携ボタン**（C案）: `/breaker`・`/pipe_size`・`/voltage_drop_v2` への値受け渡しジャンプ。実装しない
- **PDF/印刷出力**（F案）: 結果のPDF化や印刷整形。実装しない
- **住戸面積補正**（旧A案）: 内線規程に明示規定がないため初期リリースに含めない。将来要望が出れば再検討
- **戸数の拡張**（旧B案の一部）: 内線規程の表の範囲（一般1〜40 / 全電化1〜16）のみ対応。範囲外への外挿はしない
- **全電化17戸以上の対応**: 内線規程で配線用遮断器・CVTの値が示されていない範囲のため、本ツールの対象外
