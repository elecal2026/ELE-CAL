# docs/根拠 — 根拠資料 運用ルール

## 役割分け

| 場所 | 内容 | Git管理 |
|---|---|---|
| `ELE-CAL/docs/根拠/`（このフォルダ）| 検証済み抜粋MD・表データ・照合レポート | ✅ 管理対象 |
| `ELE-CAL開発フォルダ/根拠資料/内線規程/` | 内線規程 原本PDF＋全文変換MD | ❌ Git外 |
| `ELE-CAL開発フォルダ/根拠資料/公共建築/` | 公共建築 標準仕様書・標準図 原本PDF | ❌ Git外 |
| `ELE-CAL開発フォルダ/根拠資料/_rendered_pages/` | PDF目視確認用にレンダリングした対象ページPNG | ❌ Git外 |

原本PDFは最終根拠。**このフォルダのMDはPDFの抜粋・二次資料**であり、確定値扱いしない。
数値の根拠を確定する際は、必ず `根拠資料/` の原本PDFのページを目視確認すること。

---

## 各根拠MDのメタデータヘッダー仕様

各MDの冒頭には以下の7項目を必ず入れる：

```
> 出典: 内線規程 第14版 JEAC 8001-2022
> 表番号: 3705-1
> 冊子ページ: p714
> PDFページ: p796
> 対象実装: src/app/breaker/constants.ts
> 状態: OCR未確認 / 転記済み / 原本照合済み
> 照合日: YYYY-MM-DD（未照合の場合は「未照合」）
```

**冊子ページとPDFページは必ず両方記載**（ズレがある。例：冊子p714 = PDFp796）。
新規作成時の状態は必ず `OCR未確認` から始める。

---

## 根拠MD → 実装の対応

```
根拠MD（docs/根拠/）
  └─ 表データ（JSON/TS）
       └─ constants.ts / calculations.ts
```

各根拠MDの「対象実装」フィールドで、どのソースファイルに対応するかを追える。
