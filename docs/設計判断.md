# 設計判断ログ

> 「なぜそう実装したか」を1判断1セクションで記録する。
> 後で覆す場合も**古いエントリは消さない**。「却下」マークを付けて新エントリを追加。
>
> フォーマット：
> ```
> ## D-NNN: タイトル
> - **日付**: YYYY-MM-DD
> - **状態**: 採用 / 却下 / 保留
> - **背景**: どんな問題があったか
> - **判断**: 何を選んだか
> - **理由**: なぜそれを選んだか
> - **代替案**: 検討して却下した案
> - **影響**: この判断によって決まること
> ```

- **日付**: （初期実装時。正確な日付は git history 参照）

- **背景**: 一般負荷（LED照明など）の消費電力は数W単位で、kW表記だと「0.005kW」と入力する必要があり不便

## D-003: CVD/CVT許容電流の根拠表を表3（単心より合わせ）に決定

- **日付**: 2026-06-05
- **状態**: 採用
- **背景**: 資料1-3-3（JEAC8001-2022）には CVD/CVT の許容電流の根拠になり得る表が2つある。
  - **表2**（p881）: 「2心CV」「3心CV」——心線を一体に製造したケーブルの値
  - **表3**（p882）: 「単心2個より」「単心3個より」——単心CVを束ねてより合わせた状態の値
  - 表3の方が電流値が低く（保守的）、かつ現場でよく使う「CVT=単心3本をまとめて配線」の実態に近い。
  - 悩みポイント：EVケーブル/CVT の呼び方が「3心CV」を指す場合もあり、どちらの表を根拠とするか一義的に決まらなかった。
- **判断**: **表3（単心2個より/3個より）** を CVD/CVT の根拠に採用
- **理由**: ユーザー（河島さん）が実務判断として表3を選択。現場での配線実態（単心をより合わせて使う）に合致し、より保守的な値でもある。
- **代替案**: 表2（2心CV/3心CV）——電流値が高く出るため、同じ条件で選定するとケーブルが細くなる方向になる
- **影響**: allowable_current の CVD/CVT は表3値を使う。`breaker/constants.ts` の `ALLOWABLE_CURRENT` は2026-06-07に同期済み。UIラベルは現行の「CV-D（CVの2心）」「CV-T（CVの3心）」から「CV-D（単心2個より）」「CV-T（単心3個より）」へ変更が必要。

## D-002: 負荷種別の混在を許容（過去判断・変更予定）

- **日付**: （初期実装時）
- **状態**: 採用中（変更予定）
- **背景**: 1台のブレーカーに複数の負荷をぶら下げる構成を表現したい
- **判断**: 負荷リストに「一般負荷 / モーター / 溶接機」を**混在して追加可能**にした
- **理由**: 実装の単純化。「複数負荷の合算」というモデルとして自然
- **代替案**: 種別ごとに排他（最初に選んだ種別しか追加できない）
- **影響**: 現場の慣習と乖離（現場では一般負荷と溶接機を同じブレーカーに乗せない）。
  → やること一覧.md フェーズ1③で**排他選択に変更予定**
